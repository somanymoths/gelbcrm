# Workflow: от задачи до деплоя

## 1. Старт задачи

Если в первом сообщении чата есть `Новая задача`, запускается:

```bash
scripts/start-task.sh "<название задачи>" --type fix --push
```

Что делает bootstrap:
- проверяет чистоту git-дерева;
- синхронизирует `main` c `origin/main`;
- создаёт ветку `codex/<type>/<slug>`;
- создаёт task-документы в `.codex/tasks/<branch>/`;
- опционально создаёт страницу в Notion;
- поднимает dev-сервер.

## 2. ТЗ и план до кода

Перед написанием кода заполнить `.codex/tasks/<branch>/TASK.md`:
- подробное ТЗ (scope / out of scope / ограничения);
- план реализации (3-7 шагов);
- минимум 3 edge cases;
- критерии готовности.

## 3. Реализация малыми шагами

Во время реализации контролировать размер изменений:

```bash
npm run task:checkpoint
```

По умолчанию checkpoint обязателен, если:
- изменено больше 4 файлов, или
- diff больше 150 строк.

Пороги можно переопределить:

```bash
TASK_MAX_FILES=6 TASK_MAX_LINES=220 npm run task:checkpoint
```

## 4. Резюме по ходу задачи

После каждого логического этапа обновлять `.codex/tasks/<branch>/SUMMARY.md`:

```bash
npm run task:summary -- "что сделано и что осталось"
```

## 5. Финальное ревью перед PR

Собрать контекст и прогнать проверки:

```bash
npm run task:review
```

Скрипт:
- запускает `typecheck`, `lint`, `build`;
- создаёт `.codex/tasks/<branch>/FINAL_REVIEW.md`;
- фиксирует список файлов, commit-лог и чеклист финального ревью.

Ревью выполняется по трём артефактам:
- `TASK.md`;
- `SUMMARY.md`;
- `FINAL_REVIEW.md`.

## 6. PR

PR создаётся только на русском языке.

Обязательные пункты:
- соответствие исходному ТЗ/плану;
- покрытие edge cases;
- результаты проверок;
- риски и план отката.

## 7. Merge и деплой

После merge в `main`:

```bash
npm run deploy
```

Деплой всегда идёт из `origin/main` (а не из текущего состояния локальной рабочей директории).

На сервере выполняются:
- `npm ci`;
- `npm run build`;
- `npm run db:migrate`;
- рестарт сервиса (`systemd` или `pm2` fallback).
