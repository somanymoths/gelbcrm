# Раздел CRM: Преподаватели

## 1. Описание функционала

### 1.1 Цель
Раздел предназначен для управления преподавателями школы: просмотр, создание, редактирование, архивирование, восстановление и окончательное удаление (с контролем привязанных учеников).

### 1.2 Роли и доступ
1. CRUD по преподавателям доступен только роли `Админ`.
2. Остальным ролям CRUD недоступен.

### 1.3 Страницы
1. `Преподаватели` (активные).
2. `Архив преподавателей` (архивные).

### 1.4 Список активных преподавателей
1. Формат: таблица.
2. Колонки:
   - `Имя` (формат отображения: `Имя Фамилия`)
   - `Ученики` (только активные; при отсутствии показывать `0`)
   - `Контакты` (в таблице показывать только Telegram; если Telegram пустой — `Нет контакта`)
   - `Язык`
   - `Ставка`
   - `⋯` (контекстное меню)
3. Сортировка:
   - По умолчанию: `дата создания DESC` (новые сверху)
   - Пользовательская: по `Имени/Фамилии`, `кол-ву учеников`, `ставке`
4. Поиск: по `Фамилии` и `Имени`.
5. Фильтр: по `Языку`.
6. Пагинация: отсутствует.
7. Загрузка: бесконечный скролл, автоподгрузка по скроллу.
8. Размер порции: `limit=30`.
9. При ошибке догрузки: плашка ошибки + кнопка `Повторить`.
10. Метрика `Загруженность`: в текущий релиз не входит.

### 1.5 Модальное окно преподавателя
1. Открывается по клику на строку таблицы.
2. Закрывается по `Esc`, клику вне окна, кнопке `×`.
3. Содержимое:
   - Базовые данные преподавателя
   - Список учеников (только ФИО, некликабельно)
4. Действия в модалке: `Редактировать`, `Архивировать`.
5. Дополнительно действия доступны через `⋯` в строке.

### 1.6 CRUD преподавателя
1. Создание/редактирование:
   - Обязательные поля: `Фамилия`, `Имя`
   - Необязательные: `Язык` (один), `Ставка`, `Telegram`, `Телефон`, `Комментарий`
2. Архивирование:
   - Мягкое удаление (soft delete)
   - Подтверждение не требуется
   - После архивирования запись скрывается из активного списка
   - Показывается `Undo` на `9 секунд`
3. Восстановление:
   - На странице архива
4. Удаление навсегда:
   - Только на странице архива
   - Требует подтверждения
   - Блокируется при наличии любых привязанных учеников (активных/неактивных/архивных)
   - В модалке удаления показывается список привязанных учеников
   - Доступны действия:
     - `Отвязать выбранных` (чекбоксы, без подтверждения)
     - `Отвязать всех и удалить` (с подтверждением)
   - После `Отвязать выбранных`, если остались связи: показать обновлённый список, удаление оставить заблокированным
   - При частичной ошибке отвязки: преподавателя не удалять, показать неотвязанных + `Повторить`
   - Удаление навсегда только если все отвязки успешны

### 1.7 Контакты
1. Телефон: строгая маска `+7 999 999 99 99`.
2. Telegram:
   - Можно вводить ссылку или username
   - В БД хранится исходный ввод
   - В UI отображается как `@username`
3. Уникальность:
   - `Телефон` уникален, если заполнен
   - `Telegram` уникален, если заполнен
   - Проверка по всем преподавателям, включая архивных

### 1.8 Языки
1. У преподавателя выбирается один язык из справочника.
2. Новый язык добавляется в `Настройки школы -> Языки`.
3. Удаление языка запрещено, если язык используется у преподавателей.

### 1.9 Дополнительные правила
1. Дубликаты по `Фамилия + Имя` разрешены.
2. `Комментарий` — многострочный, до `1000` символов.
3. Комментарий отображается как plain text с экранированием HTML.
4. Конкурентное редактирование: `Last write wins`.
5. Toast-уведомления: `Создан`, `Сохранён`, `Архивирован` (Undo 9 сек), `Восстановлен`, `Удалён навсегда`.
6. SLA: первая загрузка списка до `2 сек` при `1000+` преподавателях.

## 2. Техническое задание

### 2.1 Границы этапа
Полная реализация: `UI + Backend API + БД миграции`.

### 2.2 Модель данных (миграции)

#### Таблица `teachers`
- `id` (PK)
- `first_name` (required)
- `last_name` (required)
- `language_id` (nullable, FK на языки)
- `rate_rub` (nullable, integer, >= 0)
- `telegram_raw` (nullable)
- `telegram_normalized` (nullable)
- `phone` (nullable)
- `comment` (nullable, до 1000 символов)
- `created_at`
- `updated_at`
- `deleted_at` (soft delete)

#### Таблица языков
Использовать существующий справочник языков школы (или создать `school_languages`, если отсутствует).

#### Связь с учениками
У ученика хранится `teacher_id` (nullable, FK на `teachers.id`).

#### Индексы и ограничения
1. Индекс на `teachers(created_at DESC)`.
2. Индекс на `teachers(last_name, first_name)`.
3. Уникальный индекс на `phone`, если не NULL.
4. Уникальный индекс на `telegram_normalized`, если не NULL.
5. Уникальность распространяется и на архивные записи.
6. Запрет удаления языка при наличии ссылок из `teachers.language_id`.

### 2.3 Валидация и нормализация
1. `first_name` и `last_name` обязательны.
2. `phone` при заполнении должен соответствовать маске `+7 999 999 99 99`.
3. `telegram_raw` может быть ссылкой, `@username` или `username`.
4. `telegram_normalized` формируется на backend для отображения и проверки уникальности.
5. `rate_rub` — целое число в рублях.
6. `comment` — максимум 1000 символов.
7. HTML в комментарии на выводе экранируется.

### 2.4 API (контракт)
1. `GET /api/teachers`
   - Query: `offset`, `limit`, `search`, `languageId`, `sortBy`, `sortDir`, `scope=active|archived`
   - Response: `items[]`, `total`, `nextOffset`
2. `POST /api/teachers`
3. `PATCH /api/teachers/{id}`
4. `POST /api/teachers/{id}/archive`
5. `POST /api/teachers/{id}/restore`
6. `DELETE /api/teachers/{id}` (удаление навсегда)
7. `GET /api/teachers/{id}/dependencies` (привязанные ученики)
8. `POST /api/teachers/{id}/unbind-students` (`studentIds[]`)
9. `POST /api/teachers/{id}/unbind-all-and-delete`
10. `GET /api/school/languages`
11. `POST /api/school/languages`
12. `DELETE /api/school/languages/{id}`
13. Все endpoint раздела преподавателей: только для роли `admin`.

### 2.5 UI/UX
1. Страница активных: таблица + поиск + фильтр языка + сортировки + infinite scroll.
2. Страница архива: таблица архивных + `Восстановить` + `Удалить навсегда`.
3. Модалка преподавателя: просмотр + редактирование + список учеников.
4. Модалка удаления: список учеников, чекбоксы, `Отвязать выбранных`, `Отвязать всех и удалить`.
5. Toast-уведомления на CRUD + Undo 9 сек для архивирования.

### 2.6 Ошибки API
1. `409 Conflict`:
   - нарушение уникальности контактов
   - удаление языка, который используется
   - удаление преподавателя при привязанных учениках
2. `422 Unprocessable Entity`: ошибки валидации.
3. `403 Forbidden`: нет прав `admin`.
4. `5xx`: общая ошибка сервера.

### 2.7 Нефункциональные требования
1. Первая загрузка списка при 1000+ преподавателях: до 2 секунд.
2. Серверная валидация обязательна.
3. Логирование ошибок и ключевых CRUD-событий.

## 4. Подробный QA-чеклист

### 4.1 Список активных
1. Таблица отображает нужные колонки.
2. Дефолтная сортировка по дате создания (новые сверху).
3. Имя отображается как `Имя Фамилия`.
4. `Ученики` считают только активных.
5. При `0` показывается `0`.
6. Контакты: Telegram приоритет; если Telegram пустой — `Нет контакта`.
7. Поиск по имени/фамилии работает корректно.
8. Фильтр по языку работает корректно.
9. Сортировки по имени/ученикам/ставке работают в обе стороны.
10. Infinite scroll догружает по 30.
11. При ошибке догрузки есть плашка + retry.

### 4.2 Модалка преподавателя
1. Открытие по клику на строку.
2. Закрытие по Esc/вне/крестик.
3. Показывает базовые поля.
4. Список учеников только текстом (ФИО).

### 4.3 Создание/редактирование
1. Сохранение только с `Имя + Фамилия`.
2. Пустые обязательные поля блокируют сохранение.
3. Комментарий >1000 символов отклоняется.
4. Телефон вне маски отклоняется.
5. Успешные операции дают корректные toast.

### 4.4 Архив/восстановление
1. Архивирование без подтверждения.
2. Undo доступен 9 секунд.
3. После Undo запись возвращается в активные.
4. В архиве работает восстановление.

### 4.5 Удаление навсегда
1. Требуется подтверждение.
2. При привязанных учениках удаление блокируется.
3. В модалке виден список привязанных.
4. `Отвязать выбранных` обновляет список оставшихся.
5. `Отвязать всех и удалить` требует подтверждения.
6. При полной успешной отвязке запись удаляется.
7. При частичной ошибке запись не удаляется, показываются неотвязанные + retry.

### 4.6 Контакты и уникальность
1. Дубликат телефона запрещён (включая архив).
2. Дубликат Telegram запрещён (включая архив).
3. Одинаковые `Имя+Фамилия` разрешены.
4. Telegram в UI отображается как `@username`.
5. Исходный Telegram хранится в БД.

### 4.7 Языки
1. Язык выбирается из справочника.
2. Добавление языка только в настройках школы.
3. Удаление используемого языка запрещено.

### 4.8 RBAC и безопасность
1. Не-админ не может выполнять CRUD.
2. Комментарий рендерится как plain text (HTML escaped).
3. Серверные валидации срабатывают при прямых API-запросах.

### 4.9 Производительность
1. При 1000+ преподавателях первая загрузка <= 2 сек.
