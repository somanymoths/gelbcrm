# Чек-лист сессии vibe coding

## 0. Перед стартом

* Открыть `plan.md`
* Выбрать только одну секцию
* Проверить scope и acceptance criteria
* Создать задачу по `task-template.md`
* Убедиться, что git state чистый

## 1. Выбор инструмента

### Когда идти в Codex

* быстрая итерация
* UI/фронтенд правки
* короткий цикл проверки

### Когда держать оба открытыми

* получить 2 варианта реализации
* одним инструментом генерировать, вторым проверять

## 2. План перед кодом

* попросить AI кратко описать план реализации
* перечислить файлы, которые будут изменены
* зафиксировать, что не должно быть затронуто
* подтвердить тестовые сценарии до кода

## 3. Сначала тесты

* определить интеграционные сценарии
* написать/обновить тесты
* убедиться, что тесты проверяют пользовательский флоу
* запустить тесты до реализации

## 4. Реализация

* один логический шаг за раз
* не просить AI достраивать весь продукт
* после заметного изменения проверять поведение руками
* следить, что AI не полез в несвязанные файлы

## 5. Ручная проверка

* happy path
* empty states
* error states
* refresh / повторный вход
* соседние сценарии не сломаны

## 6. Pre-commit gates

* lint green
* typecheck green
* integration tests green
* smoke/e2e (если есть) green
* diff проверен, мусорных изменений нет

## 7. Дисциплина commit

* отдельный commit на отдельный логический шаг
* формат: `feat|fix|test|refactor: ...`
* при риске — сначала checkpoint commit

## 8. Обновление памяти проекта

* обновить статус в `plan.md`
* добавить запись в `progress.md`
* при архитектурном выборе — запись в `decisions.md`
* при баге — запись в `bugs.md`

## 9. Если AI начинает деградировать

Признаки:

* несколько неудачных фиксов подряд
* AI переписывает лишнее
* появляются новые баги после каждого исправления

Действия:

* остановиться
* сделать checkpoint commit
* переформулировать задачу узко и конкретно
* при необходимости переключить инструмент/модель

## 10. Перед PR / релизом

* пройти `release-checklist.md`
* сверить релизный scope с `plan.md`
* убедиться, что нет открытых high/critical по релизному scope

## 11. Завершение сессии

* quality gates зелёные
* секция в `plan.md` обновлена
* есть commit
* записан следующий шаг
