# Библиотека промптов

## 1. Планирование секции

Текущая секция из `plan.md`:
[вставь секцию]

Задача:

1. Предложи четкий план реализации только этой секции
2. Перечисли конкретные файлы, которые будут созданы или изменены
3. Объясни зависимости между частями
4. Укажи риски и неясные места

Ограничения:

* не пиши код
* не трогай другие части проекта
* держи решение простым и минимальным

## 2. Критерии готовности

Дана фича:
[описание]

Сформулируй четкие проверяемые acceptance criteria:

* действия пользователя
* ожидаемый результат
* edge cases
* что не должно ломаться

## 3. Интеграционные тесты

Фича:
[описание]

Задача:

1. Предложи набор интеграционных тестов пользовательского флоу
2. Для каждого теста укажи риск, который он покрывает
3. Отметь минимальный набор для smoke

## 4. Реализация фичи

Контекст:

* секция: [название]
* тесты уже есть
* следуй `rules.md`

Требования:

* реализуй только то, что нужно для прохождения тестов
* не изменяй несвязанные файлы
* не добавляй лишние абстракции

Формат ответа:

1. Что будет реализовано
2. Какие файлы изменятся
3. Код / diff
4. Что проверить вручную

## 5. Review перед commit

Проверь реализацию.

Задача:

1. Соответствует ли criteria
2. Есть ли лишняя сложность
3. Есть ли риски регрессии
4. Что нужно поправить критично

Формат:

* findings по severity
* минимальные исправления
* готово к commit? (да/нет)

## 6. Генерация commit message

На основе изменений:
[кратко опиши]

Сгенерируй conventional commit в формате:

`type: краткое описание`

Опционально: body с ключевыми изменениями.

## 7. Анализ ошибки

Ошибка:
[текст]

Задача:

1. Назови 3-4 возможные причины
2. Отсортируй по вероятности
3. Дай способ проверки каждой

Ограничение: не писать код до выбора гипотезы.

## 8. Точечный фикс

Гипотеза причины:
[описание]

Задача: предложи минимальный фикс.

Ограничения:

* менять минимум кода
* не рефакторить несвязанные части

Формат:

* что меняется
* diff
* как проверить

## 9. Сброс подхода

Ситуация: несколько неудачных попыток.

Задача:

1. Что идет не так
2. Новый чистый подход
3. Минимальные шаги пере-реализации
